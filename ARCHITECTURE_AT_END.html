End-to-End Architecture — June 2026
Voice Booking — Request Data Flow (1 turn)
1
User Speaks
Browser captures mic via MediaRecorder (250ms chunks, audio/webm;codecs=opus). Chunks sent as binary WebSocket frames.
Client: useVoiceSession.ts
2
VAD Trigger
Chunk-size silence detection: speech ≈ ≥800B chunk. After speech + 1.5s silence → send stop_speech. Max 8s safety timer via setTimeout.
Client: chunk-size VAD
3
STT
Server concatenates all binary chunks → base64 encodes → sends to Gemini 2.5 Flash Lite multimodal. Returns {transcript, confidence}.
Google AI: gemini-2.5-flash-lite
4
Intent + Slots
stateMachine.ts runs intentDetector → slotFiller → availability check in sequence. Structured JSON output from Gemini.
Google AI: gemini-2.5-flash-lite
5
LLM Response
Gemini generates natural-language response with full session context injected. System prompt enforces all guardrails.
Google AI: gemini-2.5-flash-lite
6
TTS Stream
Response text sent to Sarvam AI. MP3 audio streamed back, chunked as base64 JSON frames over WebSocket. Browser collects all chunks then plays as single Blob.
Sarvam AI: bulbul:v3 en-IN
7
MCP Writes
On booking confirm: createTentativeHold (Calendar) + appendReservation (Sheets) via googleapis service-account auth. 1 retry on failure.
Google APIs: Calendar v3 · Sheets v4
8
Code Issued
TABLE-[A-HJ-NP-Z][00-99] generated, spoken twice in TTS. After audio ends, client fires onConfirmed → router.push('/confirmation').
Client: post-playback redirect
Component Inventory
Frontend Framework
Next.js 16.2.9 (App Router, Turbopack dev). React 19. Tailwind CSS v4.
Speech-to-Text
Gemini 2.5 Flash Lite — multimodal audio input. Batch transcription (not streaming). Returns transcript + confidence score.
LLM (Dialogue)
Gemini 2.5 Flash Lite — intent detection, slot filling, confirmation check, response generation. Structured JSON outputs via responseSchema.
Text-to-Speech
Sarvam AI bulbul:v3. Speaker: shubh. Language: en-IN. HTTP streaming to server, collected as Blob, played via HTML Audio API.
Real-time Transport
Custom WebSocket server via Node.js http.Server upgrade hook on Next.js. Binary audio frames client→server, JSON message frames server→client.
Calendar Integration
Google Calendar API v3. Tentative holds (2hr standard, 3hr large group). Service account (server-to-server auth). 1-retry wrapper.
Reservation Log
Google Sheets API v4. "Daily Reservation Log" sheet. 10-column schema. getAllReservations, appendReservation, updateStatus, lookupByCode.
Authentication
NextAuth v5 beta (App Router native). Google OAuth provider. proxy.ts protects /admin/*. JWT sessions (httpOnly cookie).
Availability Store
JSON files on disk. mockInventory.json (slot times by date/occasion). occasionConfig.json (on/off per occasion). Admin PATCH updates live.
Session State
In-memory Map on Next.js server. Keyed by sessionId. 5-minute TTL. Full SessionState interface: intent, slots, code, history, calendar/sheet IDs.
VAD
Client-side chunk-size detection in MediaRecorder.ondataavailable. Speech ≥ 800B. Silence after 1.5s post-speech triggers submit. 8s setTimeout ceiling.
Admin Portal
StatsCard · BookingsTable · AvailabilityToggle. Live data from /api/admin/reservations (Sheets) and /api/admin/availability (JSON). Optimistic toggle UI.
